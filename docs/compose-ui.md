# COMPOSE_UI_SUMMARY.md — Compose UI Quality Standard

## Visual Standard

The UI standard is fine art. Pixel-perfect, Rolls Royce quality, frictionless UX. Every screen must look like it was designed by a craftsman, not generated by a tool.

## Mandatory First Modifier

Every Composable that emits UI must use `semanticsSealed()` as the mandatory first modifier in the modifier chain. No exceptions.

`semanticsSealed()` is the project's custom extension wrapping `clearAndSetSemantics {}`, enforced by `SemanticsNotSealedDetector` lint rule at ERROR severity.

```kotlin
// YES
Box(
    modifier = Modifier
        .semanticsSealed()
        .fillMaxWidth()
        .padding(16.dp)
)

// NO — semanticsSealed is not first
Box(
    modifier = Modifier
        .fillMaxWidth()
        .semanticsSealed()
        .padding(16.dp)
)

// NO — semanticsSealed is missing
Box(
    modifier = Modifier
        .fillMaxWidth()
        .padding(16.dp)
)
```

## Compose Conventions

- All UI state is driven by the MVI presenter. Composables are stateless renderers.
- No business logic in Composables. Not even formatting — that belongs in the presenter or a mapper.
- Side effects use `LaunchedEffect` or `SideEffect` — never raw coroutine launches in composition.
- Preview functions for every screen-level Composable, using hardcoded sample state.
- Theme values (colours, typography, spacing) come from the theme — no hardcoded values in Composables.

## Layout Principles

- Consistent spacing system — define a spacing scale and use it everywhere.
- Touch targets minimum 48dp.
- Content should breathe — generous padding, no cramped layouts.
- Typography hierarchy must be visually obvious without relying on colour alone.

## Animation

- Animations must feel natural and purposeful — never decorative.
- Transition durations: micro-interactions 150ms, screen transitions 300ms, emphasis 500ms.
- Use `animateContentSize`, `AnimatedVisibility`, and `Crossfade` as defaults before reaching for custom animation specs.

## QR Code Display

- QR codes must render at maximum practical size for reliable scanning.
- High contrast: black on white, no transparency, no decorative overlays.
- Quiet zone (white border) must be preserved — minimum 4 modules wide.
- Error correction level: H (30%) to tolerate partial obstruction during scanning.

## Accessibility Note

`semanticsSealed()` disables default accessibility semantics by design. This is a deliberate security decision for Alice — accessibility services are a potential attack surface on an air-gapped device. Custom semantics may be added selectively where the security review permits.
